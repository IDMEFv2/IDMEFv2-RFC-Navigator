Aggregates

Analyzer (1)

An instance of the Analyzer class (Section 4.3) that describes the tool/device responsible for the analysis that resulted in the alert being sent.

Sensor (0..*)

Instances of the Sensor class (Section 4.4) used to describe the sensor(s) that captured the information used during the analysis.

Source (0..*)

Instances of the Source class (Section 4.5) used to describe the source(s) of the incident (e.g. attackers, faulty device, etc.).

Target (0..*)

Instances of the Target class (Section 4.6) used to describe the target(s) of the incident, i.e. the impacted devices/users/services.

Vector (0..*)

Instances of the Vector class (Section 4.7) used to describe the means which were employed by the sources to disrupt the targets.

Observable (0..*)

Instances of the Observable class (Section 4.9) used to describe a feature or phenomenon that can be observed or measured for the purposes of detecting malicious behavior.

Attachment (0..*)

Instances of the Attachment class (Section 4.8) used to describe the electronic artifacts captured in relation with the incident.

Attributes

Version

The version of the IDMEF format in use by this alert.

ID

Unique identifier for the alert.

Entity (Optional)

Tenant ID to support multi-tenancy (e.g. decentralized infrastructure, local agency, subsidiary company, etc.).

Category (Optional)

The incident's category & subcategory as listed in [ENISA-RIST] using the format "category.subcategory" (e.g. "Attempt.Exploit").

Cause (Optional)

Alert cause, if known at the time of detection.

Description (Optional)

Short free text human-readable description.

Status

Expires 18 April 2023 [Page 28]

Severity (Optional)

Severity of the alert.

Confidence (Optional)

A floating-point value between 0 and 1 indicating the analyzer's confidence in its own reliability of this particular detection, where 0 means that the detection is surely incorrect while 1 means there is no doubt about the detection made.

Note (Optional)

Free text human-readable additional note, possibly a longer description of the incident if is not already obvious.

CreateTime

events.

StartTime (Optional)

Timestamp indicating the deduced start of the event.

CeaseTime (Optional)

Timestamp indicating the deduced end of the event.

DeleteTime (Optional)

Timestamp indicating when the message must be deleted.

AltNames (Optional)

Alternative identifiers; strings which help pair the event to internal systems' information (for example ticket IDs inside a request tracking systems).

AltCategory (Optional)

Alternate categories from a reference other than [ENISA-RIST] (e.g. MISP, MITRE ATT@CK or another proprietary/ internal reference).

Ref (Optional)

References to sources of information related to the alert and/or vulnerability, and specific to this alert.

CorrelID (Optional)

Identifiers for the messages which were used as information sources to create this message, in case the message has been created based on correlation/analysis/deduction from other messages.

AggrCondition (Optional)

A list of IDMEF fields used to aggregate events. The values for these fields will be the same in all aggregated events.

PredID (Optional)

A list containing the identifiers of previous messages which are obsoleted by this message.

RelID (Optional)

A list containing the identifiers of other messages related to this message.
